# 회원가입 트래커 누락 수정 가이드

## 문제 상황

실제 회원가입 수와 간품 트래커에 기록되는 회원가입 수가 다름.

**원인:**  
회원가입 완료 시 `window.airbridge.events.send('airbridge.user.signup')`만 호출되고 있음.  
에어브릿지 이벤트를 트래커로 전달하는 `_gpWrap`이 두 스크립트(gp.js + 에어브릿지)가 **모두 로드된 이후에만** 작동하기 때문에, 사용자가 스크립트 로드 전에 가입하면 이벤트가 트래커에 전달되지 않음.

**해결:**  
회원가입 완료 시점에 `GanpoomTracker.track()`을 **직접** 호출하도록 추가.  
기존 에어브릿지 코드는 그대로 두고 한 줄만 추가하면 됨.

---

**파일 경로:**
```
ganpoomreact/src/Components/Client/SignUp.js
```

**수정 위치:** 약 183번째 줄, `if(window.airbridge)` 블록이 닫히는 `}` 바로 아래

```javascript
if(window.airbridge) {
    var snsType = ""
    if (localStorage.getItem('sns_id').charAt(0) === "K") {

    } else if (localStorage.getItem('sns_id').charAt(0) === "A") {

    window.airbridge.setUserId(localStorage.getItem('sns_id'));
    window.airbridge.setUserAttributes({ type : snsType })
    window.airbridge.events.send('airbridge.user.signup');
}

```

## 왜 이렇게 해도 기존 기능에 문제가 없나?

| 상황 | 결과 |
|---|---|
| 에어브릿지 로드됨 + GanpoomTracker 로드됨 | `_gpWrap` + 직접호출 둘 다 발동 → log.js dedup이 중복 차단, 1건만 기록 |
| 에어브릿지 로드됨 + GanpoomTracker 미로드 | `if (window.GanpoomTracker)` 조건으로 안전하게 넘어감 |
| 에어브릿지 미로드 + GanpoomTracker 로드됨 | 직접 호출로 트래커에 정상 기록 ✅ (기존엔 누락됐던 케이스) |
| 에어브릿지 미로드 + GanpoomTracker 미로드 | 아무것도 안 함, 기존과 동일 |
